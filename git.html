git 사용법 1. 인터넷에 git을 검색후 다운로드 한다.(다운로드 상세 설명은 유튜브 코딩애플 강의 참조) 2. 시작창에 터미널을 열어 "git --version"을 쳐서 깃이 깔렸는지 확인한다. 3. 깃을 사용할 대상 폴더에 powershell(그냥 터미널임, 우클릭해서 터미널 클릭)을 연다. 4. " git config --global user.email "이메일@naver.com" " 터미널에 다음과 같이 입력해 유저 이메일 등록을 한다. 5. " git config --global user.name "홍길동" " 터미널에 다음과 같이 입력해 유저 이름 등록을 한다. 6. 정보 입력을 완료했으면 애디터를 열고 해당 파일을 불러온 후에 "ctrl + shift + `"단축키로 애디터 터미널을 연다. * 중요 * 7. "git init"를 애디터 터미널에 입력해 해당 파일이 git의 영향을 받게 만든다. (이거 안 걸면 깃 시작 못 함) 8. "git add 파일명" 을 사용하면 코딩을 한 후에 만들어진 데이터 스냅샷(사진을 찍은것 처럼 데이터 형태를 저장해 어떤 이유로 코드 저장이 안되었을 시 백업의 역할을 한다.) 을 staging area에 저장할 수 있다.(staging area == 데이터가 변환되기 전에 원시/처리되지 않은 데이터가 있는 영역) 9. " git commit -m '메모할 내용' " 을 사용하면 stating area에 저장된 파일들을 repository(저장소)에 commit(맡기다) 할 수 있다. * "git status"는 내가 보낸 staging area에 있는 코딩문서가 무엇이 있는지 확인 할 수 있다. * "git log --all --oneline"는 내가 commit한 코딩문서 내역을 보여준다. 그런데 요즘엔 웬만한 애디터에 git 기능이 있어서 굳이 "git add"같은거 쓸 필요가 없다. vscode 같은 경우에도 좌측 메뉴 3번째가 깃 기능을 한다. git diff / git difftool(깃에 저장된 소스코드 비교하기) 1. "git diff"를 사용하면 git에 저장했던 코드 정보와 내가 현재 만들고 있는 코드가 무엇이 수정되고 추가되었는지 알려준다. j와 k로 스크롤바 조작이 가능하고 p로 끌 수 있다. 하지만 코드가 너무 많을 경우 확인이 힘듬 그리고 코드를 변경한 적 없는데 코드를 수정했다고 표시 될 수도 있음 그래서 사실 이거도 안 씀 2. "git difftool"을 사용하면 보다 시각적으로 훌륭한 비교창이 나온다. h / j/ k / l키로 방향이동 할 수 있다. :q!(저장 안하고 나가기)를 입력해 종료 가능하디 3. "git difftool (repository 소스코드 - "git log --all --oneline"으로 확인 가능함)"으로 입력하면 해당 소스코드랑 비교도 가능하다. 추가로 소스코드를 두 개 입력하면 그 소스코드들을 비교할 수 있다. 4. 솔직히 이렇게 터미널로 확인하는 것 보다 애디터 확장팩(좌측 메뉴 5번쨰)를 이용하면 훨씬 더 편하고 가시성 좋은 git diff를 이용할 수 있음. * 나는 git graph 확장팩 다운받아서 사용하고 있음 git branch(현재 작업물 복사본 만들기) git switch(branch 전환하기) 1. "git branch 제목"를 입력하면 파일 복사본을 만든다. 2. "git switch branch명"을 입력하면 작업파일을 해당 복사본으로 바꿔준다. 3. branch는 그냥 작업공간 정도로 생각하자. 지금 작업하고 있는 이 소스도 main branch이다. 4. 다른 branch에서 작업하다 다시 여기로 돌아오려면 "git switch main"을 입력하면 된다. * 코딩애플 행님께서 여러번 바꿔가며 commit 해보라 하셨으니 한 번 더 commit 해 봄. 5. 터미널에서 내가 저장한 commit을 확인하려 "git log --all --oneline"를 사용할때 "--graph"를 추가하면 branch 끼리 분류를 해서 보여준다. 6. "git merge branch명"을 입력하면 test에서 만든 코드가 마음에 들어서 main과 합치고 싶을때 사용한다. 7. 만약 merge를 이용하여 합칠때 중복되는 코드가 발생한다면 수동으로 수정해서 commit 해야한다. git merge 다양한기능들 1. "3-way merge" == 현재 branch와 main branch의 변경사항을 한꺼번에 merge 시켜준다.(저장하려는 branch들에 전부 commit 사항이 있을시 사용) 2. "fast-foward merge" == 해당하는 branch를 main branch로 지칭한다.(main branch에 신규 commit 사항이 없을 경우 사용) *main branch에 신규 commit이 없을 경우 자동으로 fast-forward merge가 실행된다. 3. "merge --no-ff" == fast-foward명령어가 자동으로 실행되는 것을 막고 3-way merge를 실행한다. 4. "git branch -d branch명" == merge 완료후 필요없어진 branch를 삭제할때 사용 5. "git branch -D branch명" == 실수로 만든 branch를 삭제할때 사용 rebase(merge처럼 branch 합칠때 사용) 1. "git rebase" == main branch에 신규 commit이 발생했을때 commit 이전의 main branch의 서브 branch들의 commit사항들의 내용을 commit이 발생한 main branch에게로 옮길 수 있다. (이건 코딩애플 강의에 있는 그림을 보면 이해하기 편하다.) 2. merge 하기 전에 위치를 이동해주는 명령어라 위치 변경 후에 "fast-forward merge"로 합치면 된다. 3. "3-way merge"랑 똑같은데 왜 굳이 이렇게 하냐? == 이렇게 만들면 git log 볼 때 내역이 더 깔끔하게 정리된다네 squash and merge(git log 내역 깔끔하게 정리하면서 merge 하는 방법) 1. "squash and merge" == 서브 branch 안에 있던 commit 내용들을 합쳐서 main branch의 commit으로 전환시켜준다. git revert / git reset / git restore (코드 짜다 실수헸을때 되돌아가는 명령어) 1. 파일 복구하는 법 ([] <- 무시) - [git restore "해당 파일명(ex. --.html)"]을 사용하면 컴퓨터의 ctrl + z 기능처럼 해당 파일을 마지막으로 commit 한 시점으로 되돌려준다. - [git restore --source "log에 적혀있는 commit 코드(ex. 2fc907d)" "해당 파일명(ex. --.html)"]을 사용하면 해당 commit의 시점으로 파일 전체를 되돌려준다. - [git restore --staged "해당 파일명(ex. --.html)"]을 사용하면 git add 를 시켜 staging을 해놓은 파일을 취소 시킬 수 있다. * log 중간에 끼어 있는 기록 자체를 삭제 시킬 수는 없음(과거 조작 불가능 ex. a, b, c -> a, c 불가) * 하지만 log 중간에 끼어 있는 기록에서 작업한 내용을 제거한 commit을 새로 생성 하는 것이 가능하다(ex. a, b, c -> a, b, c, d(b의 작업내용이 삭제된 내용 a, c)) - [git revert "log에 적혀있는 commit 코드(ex. 2fc907d)"]을 입력하면 해당 commit의 내용이 삭제된 새로운 commit을 만들게 된다.
