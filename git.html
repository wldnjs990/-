git 사용법 1. 인터넷에 git을 검색후 다운로드 한다.(다운로드 상세 설명은 유튜브 코딩애플 강의 참조) 2. 시작창에 터미널을 열어 "git --version"을 쳐서 깃이 깔렸는지 확인한다. 3. 깃을 사용할 대상 폴더에 powershell(그냥 터미널임, 우클릭해서 터미널 클릭)을 연다. 4. " git config --global user.email "이메일@naver.com" " 터미널에 다음과 같이 입력해 유저 이메일 등록을 한다. 5. " git config --global user.name "홍길동" " 터미널에 다음과 같이 입력해 유저 이름 등록을 한다. 6. 정보 입력을 완료했으면 애디터를 열고 해당 파일을 불러온 후에 "ctrl + shift + `"단축키로 애디터 터미널을 연다. 7. "git init"를 애디터 터미널에 입력해 해당 파일이 git의 영향을 받게 만든다. 8. "git add 파일명" 을 사용하면 코딩을 한 후에 만들어진 데이터 스냅샷(사진을 찍은것 처럼 데이터 형태를 저장해 어떤 이유로 코드 저장이 안되었을 시 백업의 역할을 한다.) 을 staging area에 저장할 수 있다.(staging area == 데이터가 변환되기 전에 원시/처리되지 않은 데이터가 있는 영역) 9. " git commit -m '메모할 내용' " 을 사용하면 stating area에 저장된 파일들을 repository(저장소)에 commit(맡기다) 할 수 있다. * "git status"는 내가 보낸 staging area에 있는 코딩문서가 무엇이 있는지 확인 할 수 있다. * "git log --all --oneline"는 내가 commit한 코딩문서 내역을 보여준다. 그런데 요즘엔 웬만한 애디터에 git 기능이 있어서 굳이 "git add"같은거 쓸 필요가 없다. vscode 같은 경우에도 좌측 메뉴 3번째가 깃 기능을 한다. git diff / git difftool(깃에 저장된 소스코드 비교하기) 1. "git diff"를 사용하면 git에 저장했던 코드 정보와 내가 현재 만들고 있는 코드가 무엇이 수정되고 추가되었는지 알려준다. j와 k로 스크롤바 조작이 가능하고 p로 끌 수 있다. 하지만 코드가 너무 많을 경우 확인이 힘듬 그리고 코드를 변경한 적 없는데 코드를 수정했다고 표시 될 수도 있음 그래서 사실 이거도 안 씀 2. "git difftool"을 사용하면 보다 시각적으로 훌륭한 비교창이 나온다. h / j/ k / l키로 방향이동 할 수 있다. :q!(저장 안하고 나가기)를 입력해 종료 가능하디 3. "git difftool (repository 소스코드 - "git log --all --oneline"으로 확인 가능함)"으로 입력하면 해당 소스코드랑 비교도 가능하다. 추가로 소스코드를 두 개 입력하면 그 소스코드들을 비교할 수 있다. 4. 솔직히 이렇게 터미널로 확인하는 것 보다 애디터 확장팩(좌측 메뉴 5번쨰)를 이용하면 훨씬 더 편하고 가시성 좋은 git diff를 이용할 수 있음. * 나는 git graph 확장팩 다운받아서 사용하고 있음 git branch(현재 작업물 복사본 만들기) 1. "git branch 제목"를 입력하면 파일 복사본을 만든다. 2. "git switch branch명"을 입력하면 작업파일을 해당 복사본으로 바꿔준다.
