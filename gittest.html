"git branch branch명"을 입력하면 다시 해당 branch로 이동할 수 있다. 코딩애플 행님께서 여러번 commit 해보라고 하셨으니 한 번 더 commit 해봄 merge 해봤는데 오류 떠서 다시 commit 해봄 숙제로 branch 새로 만들어서 다시 merge 해보라고 하셨는데 test 이거에 내용 다시 추가해서 merge 해 봄 잘 되네 ㅇㅇ -정리- 1. git branch 새로 만들고 싶다 = "git branch branch명" 2. branch 공간 바꾸고 싶다 = "git switch branch명" 3. 처음 작업했던 branch로 돌아가고 싶다 = "git switch main" 4. 특정 branch에서 작업한 소스코드가 마음에 들어서 main이랑 합치고 싶다 = "git merge branch명" (단, 모든 파일들이 commit 된 상태에서만 가능하다.) 5. merge 과정에서 코드들이 충돌된다면 직접 수동으로 수정해야 한다. * git rebase 연습중 main branch에 commit 하나 남김. * 이번엔 branch merge의 다양한 기능들을 체험해볼거임 squash and merge를 해볼거야. 먼저 이거 commit 한번 하고 한번 더 수정 사항 만들어서 commit하고 이제 main으로 switch 한 다음에 main에도 commit 사항을 하나 만들거야 * git squach 연습중 main branch에 commit 하나 남김. 일단 git rebase는 잘 됨. 그런데 commit 제목 저장을 이상하게 적어놔서 이건 바꿔야 할 듯 이건 아직 commit 안하고 squash 할 때 commit 해야겠다. 아 어짜피 새로 branch 만들어야 하는구나 그냥 commit 해야겠네 * 보니깐 rebase 다 하고 난 다음엔 "git rebase --quit"을 사용해야 나갈 수 있나봄(이건 squash라는 branch 새로 만들고 거기에 적는 문서임) 이제 squash 해보려고 함. 먼저 한번 commit 하고 * 살짝 감 잡히는게 git graph가 터미널에서 뭘 보여주는지 몰랐는데 확장팩에 나온 그래프를 보니 터미널이랑 똑같이 log 나뭇가지 친게 보이는데 확장팩이 더 가시성을 높여준거구나(터미널에서는 |, / 이런걸로 표시해주니깐 나뭇가지 모양인줄 몰랐었음) * 아무튼 squash를 하고 git log를 보니 서브 branch의 나뭇가지가 원래 merge를 할 때는 main에 연결되는 형식으로 보이는데 squash 하고 보니 서브 branch의 나뭇가지가 이어지지 않고 그 자리에서 끊기는걸 확인 할 수 있음. 나중에 나뭇가지가 막 수백가지가 만들어지면 그걸 그냥 merge만 시켜서 저장하면 보기가 정말 난잡해질 거 같으니 칠지도 마냥 중간중간 끊어서 main에만 연결시켜서 완성품만 남기는거 같다. 이제 다 쓴 suqash branch는 삭제해야징 삭제 하려 했는데 fully merge가 안됐다는데 suqash로 나뭇가지를 남겨놔서 그런건가 여기서 삭제하면 어떻게 되지 그리고 또 squash에서 commit을 해버리면 어떻게 되지 일단 squash에서 commit 한 번 더 해봐야지 squash에 commit 하니 그냥 나뭇가지가 더 길어졌음 뭐지 이러면 main에서 squash한 위치에 코드를 어떻게 알지
