"git branch branch명"을 입력하면 다시 해당 branch로 이동할 수 있다. 코딩애플 행님께서 여러번 commit 해보라고 하셨으니 한 번 더 commit 해봄 merge 해봤는데 오류 떠서 다시 commit 해봄 숙제로 branch 새로 만들어서 다시 merge 해보라고 하셨는데 test 이거에 내용 다시 추가해서 merge 해 봄 잘 되네 ㅇㅇ -정리- 1. git branch 새로 만들고 싶다 = "git branch branch명" 2. branch 공간 바꾸고 싶다 = "git switch branch명" 3. 처음 작업했던 branch로 돌아가고 싶다 = "git switch main" 4. 특정 branch에서 작업한 소스코드가 마음에 들어서 main이랑 합치고 싶다 = "git merge branch명" (단, 모든 파일들이 commit 된 상태에서만 가능하다.) 5. merge 과정에서 코드들이 충돌된다면 직접 수동으로 수정해야 한다. * git rebase 연습중 main branch에 commit 하나 남김. * 이번엔 branch merge의 다양한 기능들을 체험해볼거임 squash and merge를 해볼거야. 먼저 이거 commit 한번 하고 한번 더 수정 사항 만들어서 commit하고 이제 main으로 switch 한 다음에 main에도 commit 사항을 하나 만들거야 * git squach 연습중 main branch에 commit 하나 남김. 일단 git rebase는 잘 됨. 그런데 commit 제목 저장을 이상하게 적어놔서 이건 바꿔야 할 듯 이건 아직 commit 안하고 squash 할 때 commit 해야겠다. 아 어짜피 새로 branch 만들어야 하는구나 그냥 commit 해야겠네 * 보니깐 rebase 다 하고 난 다음엔 "git rebase --quit"을 사용해야 나갈 수 있나봄(이건 squash라는 branch 새로 만들고 거기에 적는 문서임) 이제 squash 해보려고 함. 먼저 한번 commit 하고
